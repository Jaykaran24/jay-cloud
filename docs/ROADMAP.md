# Vision

Jay Cloud aims to become a complete self-hosted cloud platform that enables users to deploy applications, manage infrastructure, monitor services, store files, and access personal projects from a single unified dashboard.

The platform combines the functionality of multiple tools into one cohesive experience.

Long-term inspiration:

- Vercel
- Railway
- Render
- Portainer
- Google Drive
- Grafana
- Uptime Kuma

The goal is **not** to copy these platforms, but to create a unified developer experience inspired by their best ideas.

---

# Development Principles

Development follows these principles:

- Build vertically, not horizontally.
- Complete one feature before moving to the next.
- Every milestone must produce a usable product.
- Prioritize stability over feature count.
- Every feature must improve the user experience.

# Success Criteria

Jay Cloud will be considered production-ready when:

- Public website is complete.
- Authentication is secure.
- Docker management is stable.
- Storage is reliable.
- Monitoring is accurate.
- Documentation is complete.
- Platform supports multiple users.
- All core infrastructure can be managed from Jay Cloud.
